TECHNICAL SPECIFICATION: COMPACT COUNTER
================================================

1. ARCHITECTURE STRATEGY
------------------------
This implementation uses a "Compact Inline" style. Logic is 
embedded directly within HTML attributes to reduce the 
overall file size and keep the script block minimal.

2. VARIABLE DEFINITIONS
-----------------------
In this code, identifiers are shortened for brevity:

- Variable [s]: 
  Represents the DOM Interface. It captures the <h1> 
  element using document.getElementById. This is the 
  "Output Channel" where the user sees the number.

- Variable [a]: 
  Represents the "State" or "Accumulator." This is the 
  actual number being tracked in the computer's memory.

3. LOGIC & LINE BREAKS
----------------------
You will notice the code inside the 'onclick' attributes 
is spread across multiple lines:

Example:
   a=a+1
   s.innerText=a

REASON: JavaScript requires a separator between commands. 
While a semicolon (;) is standard, a line break (New Line) 
also acts as a command terminator. In this compact file, 
line breaks are used to keep the logic readable without 
relying on semicolons.

4. COMPONENT FUNCTIONALITY
--------------------------
The application follows a "Trigger-Modify-Update" pattern:

1. TRIGGER: User clicks a <button>.
2. MODIFY: The variable [a] is recalculated:
   - sub:   a = a - 1
   - reset: a = 0
   - add:   a = a + 1
3. UPDATE: The new value of [a] is immediately pushed 
   to the innerText of [s].

5. USAGE WARNING
----------------
This is a "Volatile Counter." Because the state is stored 
in a local JavaScript variable [a], refreshing the browser 
tab will wipe the memory and return the counter to 0.
